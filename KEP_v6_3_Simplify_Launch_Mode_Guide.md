# KEP v6.3 — Simplify & Launch Mode Guide

## Purpose

KEP has enough features for the first testing stage.

From now, the smart move is to simplify.

## Student side

Students should mainly see:

- Start
- Practice
- Mock
- Progress
- Report

That is enough for first testing.

## Admin side

Admins/reviewers should mainly see:

- Review
- Quality
- Feedback

AI and developer tools should not be the main visible experience.

## Developer side

Developer/setup pages still exist, but they stay separate:

- Setup
- Database
- Content
- Deploy

## What v6.3 changed

- Added `start.html`.
- Simplified public navigation.
- Simplified student navigation.
- Simplified admin navigation.
- Added a clearer student journey on the homepage.
- Kept feedback and report pages available.
- No new database table.
- No new SQL required.

## Recommended next action

Use v6.3 as the first clean version for trusted testers.

Do not add more big features until testers give feedback.
